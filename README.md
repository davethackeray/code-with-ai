### The Thoughtful Builder Starter Kit
You're already learning to code. You just don't know it yet.

This repo gives you two files that turn vibe coding into an apprenticeship. No course to finish before you start. No theory without context. You build. You learn. At the same time.

### The Five Rules
1. Start with the project, not the curriculum.
You learn faster when every lesson has a destination. Fork this repo, describe what you're building in LEARNING.md, and begin.

2. The pause is the classroom.
Every gap between your prompt and the AI's output is a teaching moment. The AI will use it. Let it.

3. The student is not stupid. They are contextless.
LEARNING.md fixes the context. It tells the AI what you know, what you've struggled with, and what you're building. Update it honestly.

4. Understand the architecture, not just the output.
You don't need to read every line of code. You need to understand how the whole thing fits together. The Three Altitudes section in LEARNING.md is where that understanding lives.

5. The sinking ship is the syllabus.
Every mistake is a lesson. Log it in LEARNING.md. The AI will reference it. You won't make it twice.

### What's in this repo
LEARNING.md — Your living learning record. The AI reads this every session.

.cursorrules — Instructions for Cursor. Makes the AI behave as a tutor, not just a contractor.

SYSTEM_PROMPT.md — Same as .cursorrules, for use with Gemini, Claude, ChatGPT, or any AI without a rules file. Paste it as your system prompt.

### How to use it
1. Fork this repo
2. Open it in Cursor (or your AI editor of choice)
3. Fill in the "What I'm Building" section of LEARNING.md
4. Start building

That's it.

Works with: Cursor + any model (Claude, GPT-4o, Gemini), Windsurf, any AI with a system prompt (use SYSTEM_PROMPT.md).
